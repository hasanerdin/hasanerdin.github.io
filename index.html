🎮 2048 Clone in Unity – Project Overview

During the development of my 2048 clone using Unity, I focused not only on replicating the game's core mechanics, but also on applying and learning essential Unity concepts that are widely used in professional game development. This project helped me structure code more modularly, organize the UI efficiently, and understand deeper event-driven systems.

🔧 Tools & Systems Used

  • 🎨 Asset Importing & TextMeshPro — Used for sharp, scalable text and UI display.
  • 🧱 Canvas & UI Layouts — Responsive UI via Vertical and Horizontal Layout Groups.
  • 📦 Prefab System — Tiles are prefabs, spawned and updated dynamically.
  • 📄 ScriptableObject — Stores tile data and configuration cleanly.

🎮 Gameplay Architecture

  • 📲 Input System — Keyboard + mobile swipe detection using EventHandlers.
  • 🔁 Coroutines — Smooth animations and UI delays.
  • 🚦 Game States — Includes Start and Game Over screens with logic separation.
  • 🕹️ UI Buttons — Restart and control buttons connected via UnityEvents.

🧠 What I Learned

  • Scalable UI design with Unity Layouts
  • Prefab-driven architecture
  • ScriptableObject usage for clean data separation
  • Event-driven input handling
  • Basic mobile swipe support
  • Coroutines for animation timing
  • Game state transitions and UI control
