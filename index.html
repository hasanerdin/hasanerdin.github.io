🎮 2048 Clone in Unity – Project Overview

During the development of my 2048 clone using Unity, I focused not only on replicating the game's core mechanics, but also on applying and learning essential Unity concepts that are widely used in professional game development. This project helped me structure code more modularly, organize the UI efficiently, and understand deeper event-driven systems. The project was built by following and expanding upon this tutorial video

🔧 Tools & Systems Used

  • 🎨 Asset Importing & TextMeshPro — Used for sharp, scalable text and UI display.
  • 🧱 Canvas & UI Layouts — Responsive UI via Vertical and Horizontal Layout Groups.
  • 📦 Prefab System — Tiles are prefabs, spawned and updated dynamically.
  • 📄 ScriptableObject — Stores tile data and configuration cleanly.

🎮 Gameplay Architecture

  • 📲 Input System — Keyboard + mobile swipe detection using EventHandlers.
  • 🔁 Coroutines — Smooth animations and UI delays.
  • 🚦 Game States — Includes Start and Game Over screens with logic separation.
  • 🕹️ UI Buttons — Restart and control buttons connected via UnityEvents.

🧠 What I Learned

  • Scalable UI design with Unity Layouts
  • Prefab-driven architecture
  • ScriptableObject usage for clean data separation
  • Event-driven input handling
  • Basic mobile swipe support
  • Coroutines for animation timing
  • Game state transitions and UI control

🟩 Wordle Clone in Unity – Project Overview

During the development of my Wordle clone in Unity, I recreated the core gameplay of the popular word puzzle while exploring Unity’s UI system, prefab architecture, and data-driven design. This project allowed me to combine UI layouts, input handling, and logic scripting into a complete, functional game. The project was built by following and expanding upon this tutorial video .

🔧 Tools & Systems Used

  • 🎨 Canvas & UI Layouts — Built the letter grid using Grid Layout Groups for a responsive layout.
  • 📦 Prefab System — Letter tiles were prefabs, dynamically instantiated and updated at runtime.
  • 📄 Resources — Stored and loaded the word list from Unity’s Resources folder.
  • ⌨ Input Handling — Captured keyboard input for letter entry and submission.

🎮 Gameplay Architecture

  • 📝 Word Validation — Compared player guesses against a predefined word list.
  • 🎯 Tile Feedback — Updated tile colors based on letter correctness (correct, misplaced, or wrong).
  • 🚦 Game States — Managed win/lose conditions and prevented further input after game end.
  • 💡 Feedback System — Displayed messages for invalid words and game results.

🧠 What I Learned

  • Using Unity’s Grid Layout Groups for dynamic UI design
  • Prefab instantiation and runtime updates
  • Loading and parsing data from the Resources folder
  • Implementing a simple input and validation system
  • Managing game flow through state control
